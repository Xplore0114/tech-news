## 二、页面结构重设计

### 当前结构（5个Tab）
```
国际新闻 | 国内新闻 | 实时热点 | 科技股价 | 历史日报
```

### 建议新结构（6个Tab）

```
📰 今日速览 | 🌍 国际 | 🇨🇳 国内 | 🔥 热点 | 📈 行情 | 📚 归档
```

#### Tab 说明

**📰 今日速览（新增）**
- 作为首页/默认落地页
- 聚合当天最重要的内容：国际TOP3 + 国内TOP3 + 热点TOP3 + 股价涨跌幅榜TOP3
- 一屏看完全天重点，不需要切换Tab
- 底部显示"完整内容请切换对应Tab"

**🌍 国际**
- 保持现有 Hacker News + GitHub Trending 内容
- 新增来源标签颜色区分（HN=蓝，GitHub=绿）

**🇨🇳 国内**
- 保持现有掘金内容
- 新增 36氪、虎嗅、澎湃新闻（已在待办列表）

**🔥 热点**
- 微博热搜 + 知乎热榜
- 当前因反爬暂无数据，GitHub Actions 环境应正常

**📈 行情（原科技股价）**
- 修复股价数据源（见第三节）
- 新增 A股科技板块（可选）
- 黄金/白银保留

**📚 归档（原历史日报）**
- 保持现有功能，改名更清晰

## 三、股价数据源修复方案

### 当前问题
Yahoo Finance 通过 CORS 代理拉取不稳定，大多数 symbol 返回失败。

### 建议方案：Finnhub API

- 官网：https://finnhub.io
- 免费额度：60次/分钟，足够
- **原生支持 CORS**，浏览器直接调用无需代理
- 支持美股、港股（需要 `HK:` 前缀）

#### 需要的操作
1. 注册 finnhub.io 获取免费 API Key（约30秒）
2. 将 API Key 配置到页面（硬编码或通过 GitHub Actions 注入）

## 四、视觉设计调整

### 当前问题
- 深色主题整体不错，但 Header 渐变不够有辨识度
- Tab 栏在移动端文字过小
- 卡片信息密度偏低（空白较多）

### 建议调整

1. **Header**：加入当前日期大字显示 + 今日新闻总数统计
2. **Tab 图标**：每个 Tab 加 emoji 前缀（已在新结构中体现）
3. **卡片**：来源标签加颜色区分，不同来源用不同色
4. **股价卡片**：涨跌幅用更大字体，更直观
5. **移动端**：Tab 栏支持左右滑动（已有 overflow-x:auto，保持）
